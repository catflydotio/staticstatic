Second Post

I have a couple of pages now in this tediously-typed-in static HTML site. Of course I'm starting to miss having some code to put it all together for me. Being uncomfortable, and remembering things that static site generators or web frameworks do for us, is part of the point.

Now that there are two posts, it's very tempting to put at least a "previous" button in here. How much complexity should I build in here before starting to think about how I should automate it? What's the next post? What's the rest of this post??

Well, here's one thought: I'd like to work out a version of what my templates should look like, before turning to software to implement them. This is a solid reason to resist skipping to solutions for automation.

So what did I end up changing?

  • Added a "previous" button. No "next" button until I write another entry, though.
  • Also: I created a "page" at http://localhost:8000/pages/index-screenshots.html, where I can collect screenshots of what my index page looks like at any given time. Do I need this? Not sure, but if I keep changing things, I may be interested to see a record of how they looked at a given point. And I can't go back in time to start keeping a record. So this is there to be deleted if needed.

Rubber-ducking

  • Why is an unstyled HTML document so painful to look at?

    It's really not so bad if I narrow my browser window. I let that simmer for a bit and concluded that taking the edge off the white background and adding margins to left and right would do wonders. Something to keep in mind when I add a stylesheet!

    Total duh, but gimme a break. I'm trying to be systematic.

  • What goes in the next post? I'm running out of clear purpose already! I don't cave and add a stylesheet already, do I?

    No, I don't think so. A screen reader user with total blindness is not going to see my styling, so the page has to be as complete and comprehensible as possible without it. If I add CSS as I go, I'll likely be distracted by it and let semantic wooliness creep in.

    So, the next thing is to add something to the DOM.
